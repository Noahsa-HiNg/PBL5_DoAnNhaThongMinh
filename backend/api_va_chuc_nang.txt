# DANH SÁCH API VÀ CHỨC NĂNG DỰ KIẾN - DỰ ÁN PBL5 SMART HOME EDGE AI

Dựa trên cấu trúc hệ thống (từ README.md) và cấu trúc dữ liệu cơ sở dữ liệu (từ database_repo.py), dưới đây là danh sách phân tích các chức năng và API RESTful (`FastAPI`) dự kiến cần xây dựng cho dự án.

---

## 1. CÁC TÍNH NĂNG CHÍNH CỦA HỆ THỐNG
1. **Quản lý danh mục (Master Data Management):**
   - Quản lý các phòng (Phòng Khách, Phòng Ngủ, v.v.).
   - Quản lý loại thiết bị (Nhiệt độ, Độ ẩm, Đèn, Quạt).
   - Quản lý các node điều khiển (ESP32).
2. **Quản lý thiết bị (Device Management):**
   - Thêm, sửa, xóa, lấy danh sách thiết bị.
   - Gắn thiết bị vào các phòng, board mạch (ESP32), và chọn chân (pin) điều khiển cụ thể.
3. **Giám sát trạng thái & Điều khiển (Status & Control):**
   - Bật/Tắt thiết bị (Actuator: Đèn, Quạt).
   - Theo dõi trạng thái thiết bị hiện hành (On/Off).
4. **Theo dõi Cảm biến (Sensor Monitoring):**
   - Thu thập dữ liệu từ cảm biến gửi về.
   - Trích xuất lịch sử thông số cảm biến (nhiệt độ, độ ẩm).
5. **Điều khiển bằng Giọng nói AI (Voice Control Offline):**
   - Xử lý ngôn ngữ tự nhiên (NLP) phân tích câu lệnh tiếng Việt thành lệnh điều khiển thực tế.
6. **Camera Stream:**
   - Cung cấp luồng video trực tiếp từ máy ảnh trên Raspberry Pi.

## 2. DANH SÁCH API ĐỀ XUẤT (BACKEND FASTAPI)

### 2.1. API - Quản lý Phòng (Rooms)
| Method | Endpoint | Description (Chức năng) |
| :--- | :--- | :--- |
| `GET` | `/api/rooms` | Lấy danh sách toàn bộ phòng trong nhà. |
| `POST` | `/api/rooms` | Tạo một phòng mới. |
| `PUT` | `/api/rooms/{id}` | Cập nhật thông tin/tên phòng. |
| `DELETE` | `/api/rooms/{id}` | Xóa một phòng khỏi hệ thống. |

**Dữ liệu mẫu:**
- `POST /api/rooms` & `PUT /api/rooms/{id}` (Body):
  ```json
  {
    "name": "Phòng Khách"
  }
  ```
- Response (Thành công):
  ```json
  { "message": "Tạo phòng thành công!" }
  ```

### 2.2. API - Quản lý ESP Nodes (Mạch điều khiển)
| Method | Endpoint | Description (Chức năng) |
| :--- | :--- | :--- |
| `GET` | `/api/nodes` | Lấy danh sách các board mạch ESP32. |
| `POST` | `/api/nodes` | Thêm board ESP32 mới (kèm địa chỉ MAC, vị trí). |
| `PUT` | `/api/nodes/{id}` | Chỉnh sửa thông tin node ESP32. |
| `DELETE` | `/api/nodes/{id}` | Xóa khai báo về board ESP32. |

**Dữ liệu mẫu:**
- `POST /api/nodes` & `PUT /api/nodes/{id}` (Body):
  ```json
  {
    "mac_address": "AA:BB:CC:DD:EE:FF",
    "location_desc": "Tủ điện Tầng 1"
  }
  ```

### 2.3. API - Quản lý Danh mục Loại thiết bị (Device Types)
| Method | Endpoint | Description (Chức năng) |
| :--- | :--- | :--- |
| `GET` | `/api/types` | Lấy danh sách phân loại (Cảm biến/Công tắc - Sensor/Actuator). |

### 2.4. API - Quản lý Thiết bị (Devices)
| Method | Endpoint | Description (Chức năng) |
| :--- | :--- | :--- |
| `GET` | `/api/devices` | Lấy danh sách toàn bộ thiết bị (kèm thông tin phòng, loại, node). |
| `GET` | `/api/devices/{id}` | Xem chi tiết cấu hình 1 thiết bị. |
| `POST` | `/api/devices` | Đăng ký một thiết bị mới (Nối Sensor/Relay vào ESP). |
| `PUT` | `/api/devices/{id}` | Cập nhật tên, chân pin, việc gắn phòng của thiết bị. |
| `DELETE` | `/api/devices/{id}` | Xóa thiết bị. |

**Dữ liệu mẫu:**
- `POST /api/devices` (Body):
  ```json
  {
    "name": "Đèn chính PK",
    "room_id": 1,
    "type_id": 3,
    "esp_id": 1,
    "pin": 5
  }
  ```
- `PUT /api/devices/{id}` (Body - Các trường đều Optional):
  ```json
  {
    "name": "Đèn trần PK",
    "pin": 6
  }
  ```

### 2.5. API - Trạng thái & Điều khiển (Status & Control)
| Method | Endpoint | Description (Chức năng) |
| :--- | :--- | :--- |
| `GET` | `/api/devices/{id}/status` | Truy vấn trạng thái hiện tại (ON/OFF) của 1 Actuator. |
| `POST` | `/api/devices/{id}/control` | **API Quan trọng:** Lệnh người dùng (từ Web) bật/tắt thiết bị. (Gửi kèm `{"status": "ON"|"OFF"}`). API này sau đó sẽ publish lệnh qua MQTT xuống ESP32. |

**Dữ liệu mẫu:**
- `POST /api/devices/{id}/control` (Body):
  ```json
  {
    "status": "ON"
  }
  ```

### 2.6. API - Dữ liệu Cảm biến (Sensor Data)
| Method | Endpoint | Description (Chức năng) |
| :--- | :--- | :--- |
| `POST` | `/api/sensors/data` | **API Nhận Dữ Liệu:** ESP32/MQTT Worker gọi để lưu thông số nhiệt độ/độ ẩm mới vào DB. |
| `GET` | `/api/sensors/history` | Truy xuất lịch sử cảm biến (có thể lọc theo khoảng thời gian, theo `device_id`) để vẽ biểu đồ trên Dashboard. |

**Dữ liệu mẫu:**
- `POST /api/sensors/data` (Body):
  ```json
  {
    "device_id": 10,
    "value": 26.5
  }
  ```

### 2.7. Luồng dữ liệu qua giao thức MQTT (Non-HTTP)
Ngoài API chuẩn (REST), hệ thống cần các MQTT Topics cho chức năng Realtime:
- Lệnh từ Server xuống ESP: `smarthome/control/<mac_address>` (Payload: `{pin: 5, state: 1}`).
- Dữ liệu cảm biến từ ESP lên Server: `smarthome/sensor/<mac_address>`.
- Khai báo/Heartbeat từ ESP báo lên Server để báo "Tôi đang online".

### 2.8. API - AI & Camera
| Method | Endpoint | Description (Chức năng) |
| :--- | :--- | :--- |
| `GET` | `/api/camera/stream` | Stream luồng hình ảnh MJPEG từ camera (Pi Camera V2). |
| `POST` | `/api/voice/command` | Nhận một file text, hoặc file âm thanh đã chuyển thành text, phân tích ý định (Intent) tiếng Việt (vd "bật quạt đi") bằng mô hình JointBERT để tự động kích hoạt API điều khiển ở phần 2.5. |

**Dữ liệu mẫu:**
- `POST /api/voice/command` (Body):
  ```json
  {
    "text": "Bật đèn phòng khách lên cho sáng"
  }
  ```
